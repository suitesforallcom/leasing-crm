# Stripe — простая пошаговая инструкция

Эта инструкция для человека, который никогда не работал с терминалом.
Делай шаги ровно в таком порядке. Не спеши. Если что-то пошло не так —
не паникуй, внизу есть раздел «Что делать если не получилось».

---

## Что мы делаем и зачем

Сейчас твоя программа умеет ставить крестики напротив месяцев (paid / late).
Мы учим её разговаривать со Stripe — это значит:

- Ты жмёшь кнопку «Send Invoice» — Stripe сам отправляет арендатору email
  со счётом.
- Арендатор жмёт «Pay» в этом email — деньги идут на твой Stripe счёт.
- Stripe сам рассказывает нашей программе «деньги пришли» — и зелёный
  кружок «paid» сам появляется в таблице Payments.

Чтобы это заработало, нам нужно подружить две системы: твой проект Firebase
(где живёт сайт) и Stripe. Эта инструкция — именно про эту «дружбу».

Всё делается на твоём Mac через специальное окно, которое называется
«Терминал». Ничего страшного там нет — это просто чёрное окно, куда
копируют команды.

---

## Подготовка — что должно быть под рукой

- Mac, включённый и подключённый к интернету.
- Доступ к email на который зарегистрирован твой Firebase проект
  (tony@al-en.com — он же Google аккаунт).
- Доступ к Stripe аккаунту (логин + пароль).
- Примерно 30 минут свободного времени.

## ШАГ 1. Открыть Терминал

**Что это:** чёрное окно для ввода команд. Мы будем копировать команды из
этой инструкции и вставлять их туда.

**Как открыть:**

1. На клавиатуре нажми одновременно `Cmd` (кнопка с яблоком) + `Space` (пробел).
2. Появится маленькое окошко поиска в центре экрана.
3. Напечатай слово `Terminal` и нажми Enter.
4. Откроется чёрное (или белое) окно. Это и есть Терминал.

**Как выглядит:**
В окне будет что-то вроде `MacBook-Pro:~ tony$` или `tony@MacBook ~ %`.
Это нормально. Курсор мигает — можно печатать.

**Важное правило:** после каждой команды обязательно жми Enter.
Без Enter команда не выполнится.

---

## ШАГ 2. Проверить что установлен Node.js

Node.js — это технология на которой работает Firebase CLI. Без неё
дальше не получится.

**Скопируй и вставь в Терминал эту команду:**

```
node --version
```

Нажми Enter.

**Что ты должен увидеть:**
Что-то вроде `v20.11.0` или `v18.17.0`. Любая версия от v18 и выше —
нормально.

**Если увидишь ошибку** `command not found: node` — значит Node.js не
установлен. Открой в браузере [https://nodejs.org](https://nodejs.org),
скачай версию с надписью «LTS» (большая зелёная кнопка слева), установи
её (два клика Next-Next-Next), потом перезапусти Терминал и снова набери
`node --version`.

---

## ШАГ 3. Установить Firebase CLI

**Что это:** программа которая позволяет твоему Терминалу общаться с
Firebase. Это надо сделать один раз — потом она будет работать всегда.

**Скопируй в Терминал:**

```
npm install -g firebase-tools
```

Нажми Enter.

**Что ты должен увидеть:**
Будет крутиться пару минут. В конце появится что-то вроде
`added 450 packages in 2m`.

**Возможные ошибки:**
- Если написано `permission denied` — попробуй команду с `sudo` в начале:
  `sudo npm install -g firebase-tools`. Mac спросит пароль (тот же что
  вводишь когда включаешь компьютер). Когда печатаешь пароль — буквы не
  показываются, это нормально.

---

## ШАГ 4. Проверить что Firebase CLI установился

**Скопируй в Терминал:**

```
firebase --version
```

Нажми Enter.

**Что ты должен увидеть:** номер версии, например `13.29.1`. Если
появился номер — всё хорошо, идём дальше.

---

## ШАГ 5. Войти в Firebase

Мы должны сказать Терминалу: «я — это Tony, вот мой Google аккаунт».

**Скопируй в Терминал:**

```
firebase login
```

Нажми Enter.

**Что произойдёт:**
1. Терминал спросит разрешение собирать анонимную статистику. Напиши `Y`
   или `N` (без разницы) и нажми Enter.
2. Автоматически откроется твой браузер на странице входа Google.
3. Выбери аккаунт `tony@al-en.com`.
4. Разреши Firebase доступ (жми Allow / Разрешить).
5. Браузер покажет страницу `Firebase CLI Login Successful` — можно
   закрыть.
6. В Терминале появится `Success! Logged in as tony@al-en.com`.

**Если на втором запуске пишет `Already logged in` — всё хорошо, ты
уже залогинен.**

## ШАГ 6. Перейти в папку проекта

Терминал сейчас «находится» в домашней папке. Нам нужно попасть в папку
с твоим проектом.

**Скопируй в Терминал (в одну строку!):**

```
cd "/Users/diskc/Documents/Claude/Projects/Office map"
```

Нажми Enter.

**Что ты должен увидеть:**
Строка в Терминале поменяется — теперь там будет что-то вроде
`Office map %` или `Office map $`. Это значит мы в нужной папке.

**Важно:** кавычки обязательны, потому что в названии папки есть пробел.

---

## ШАГ 7. Привязать папку к Firebase проекту

Мы говорим Firebase: «эта папка относится к такому-то проекту в Firebase».

**Скопируй в Терминал:**

```
firebase use --add
```

Нажми Enter.

**Что ты увидишь:**
Появится список твоих Firebase проектов. Что-то вроде:

```
? Which project do you want to add?
> suitesforall-prod
  my-other-project
```

Стрелками вверх/вниз выбери свой основной проект (тот где работает сайт).
Нажми Enter.

Потом появится:
```
? What alias do you want to use for this project?
```

Напиши `default` и нажми Enter.

**Готово — папка привязана к проекту.**

## ШАГ 8. Получить Stripe Secret Key

Теперь нам нужен «секретный ключ» от Stripe — это как пароль, по которому
наша программа будет общаться со Stripe.

**ВАЖНО:** этот ключ никому не показывай. Его нельзя пересылать в чаты,
писать в email, постить в соцсети. Если случайно показал — немедленно
иди в Stripe и нажми «Roll key» чтобы сгенерировать новый.

**Как получить:**

1. Открой в браузере: [https://dashboard.stripe.com/apikeys](https://dashboard.stripe.com/apikeys)
2. Войди в свой Stripe аккаунт если не был залогинен.
3. Убедись что вверху страницы написано **«Live mode»** (не Test mode).
   Это переключатель справа вверху.
4. Найди раздел **Standard keys**.
5. Там будут две строки:
   - **Publishable key** (начинается с `pk_live_...`) — НЕ ЭТА
   - **Secret key** (начинается с `sk_live_...`) — ВОТ ЭТА
6. Справа от Secret key нажми кнопку **Reveal live key**.
7. Откроется окно с длинной строкой вроде
   `sk_live_51H8aBcD...` (длиной символов 100+).
8. Нажми на значок копирования справа от ключа (квадрат с квадратом).

**Ключ сейчас у тебя в буфере обмена.** Идём дальше.

---

## ШАГ 9. Сохранить Stripe ключ в Firebase

**Скопируй в Терминал:**

```
firebase functions:secrets:set STRIPE_SECRET_KEY
```

Нажми Enter.

**Что ты увидишь:**
```
? Enter a value for STRIPE_SECRET_KEY
```

Сейчас курсор ждёт пока ты вставишь ключ. Нажми `Cmd + V` (Paste).
**Ты не увидишь что вставилось — это нормально.** Терминал специально
прячет секреты чтобы никто не подсмотрел через плечо.

Нажми Enter.

**Что ты увидишь:**
```
Secret STRIPE_SECRET_KEY created successfully
```

Всё, ключ в безопасности — он хранится в Google Secret Manager,
зашифрован, и никто кроме Cloud Functions его не увидит.

## ШАГ 11. Первый деплой

Деплой — это «отправить код на Firebase сервера чтобы он там начал работать».

**Скопируй в Терминал:**

```
firebase deploy --only functions
```

Нажми Enter.

**Что ты увидишь:**
Будет крутиться 2–5 минут. Терминал напишет разные сообщения. В конце:

```
+ Deploy complete!

Function URL (ping(us-central1)):
  https://us-central1-ТВОЙ-ПРОЕКТ.cloudfunctions.net/ping

Function URL (stripeWebhook(us-central1)):
  https://us-central1-ТВОЙ-ПРОЕКТ.cloudfunctions.net/stripeWebhook
```

**КРИТИЧЕСКИ ВАЖНО:** скопируй вторую строку (`stripeWebhook`) в отдельное
место — например в Notes на Mac или прямо в заметки Stickies. Она нам
нужна через минуту.

## ШАГ 12. Проверить что функции работают

**Скопируй в Терминал (подставь свой URL из шага 11):**

```
curl https://us-central1-ТВОЙ-ПРОЕКТ.cloudfunctions.net/ping
```

Нажми Enter.

**Что ты должен увидеть:**
```
{"ok":true,"service":"suitesforall-functions","time":"2026-04-19T..."}
```

**Если увидел `{"ok":true...}` — ура, функции работают!**

Если увидел `502` или `Error` — подожди 30 секунд и повтори команду.
Функции после деплоя иногда «просыпаются» не сразу.

## ШАГ 13. Создать webhook в Stripe

Webhook — это «труба» по которой Stripe будет звонить нашим функциям и
сообщать «этот платёж прошёл».

**Пошагово:**

1. Открой [https://dashboard.stripe.com/webhooks](https://dashboard.stripe.com/webhooks)
2. Проверь что вверху страницы **Live mode** (не Test mode).
3. Нажми большую синюю кнопку **Add endpoint**.
4. Появится форма с полями:
   - **Endpoint URL** — вставь URL `stripeWebhook` из шага 11
     (то что заканчивается на `/stripeWebhook`)
   - **Description** — напиши `SuitesForAll invoice events`
5. Прокрути вниз до **Select events**.
6. Нажми **+ Select events**.
7. В поиске введи `invoice.payment_succeeded` — поставь галочку.
8. Очисти поиск, введи `invoice.payment_failed` — поставь галочку.
9. Очисти поиск, введи `customer.subscription.created` — поставь галочку.
10. Очисти поиск, введи `customer.subscription.updated` — поставь галочку.
11. Очисти поиск, введи `customer.subscription.deleted` — поставь галочку.
12. Внизу нажми **Add events**.
13. Внизу нажми **Add endpoint** (синяя кнопка).

**Готово — webhook создан.**

Теперь тебе нужен **Signing secret**:

14. На странице свежесозданного webhook ищи раздел **Signing secret**.
15. Нажми **Reveal** или **Click to reveal**.
16. Появится строка вида `whsec_abc123...` — скопируй её (значок копирования).

## ШАГ 16. Финальная проверка

**Тест 1 — ping всё ещё работает:**

```
curl https://us-central1-ТВОЙ-ПРОЕКТ.cloudfunctions.net/ping
```

Должен вернуть `{"ok":true,...}`.

**Тест 2 — Stripe видит webhook:**

1. Открой [https://dashboard.stripe.com/webhooks](https://dashboard.stripe.com/webhooks)
2. Нажми на свой webhook (созданный в шаге 13).
3. Вверху справа нажми **Send test webhook**.
4. В выпадашке выбери `invoice.payment_succeeded`.
5. Нажми **Send test webhook**.
6. Появится зелёная плашка **Test webhook sent**.
7. Через 10 секунд обнови страницу — в разделе **Events** должно появиться
   твоё тестовое событие с зелёной галочкой и статусом **200** или `Succeeded`.

**Если статус 200 — всё работает!** Если не 200 — пиши мне какой статус
и какое сообщение об ошибке, разберёмся.

## Что делать если не получилось

### «command not found»
Значит предыдущий шаг не завершился успешно. Вернись на один шаг назад
и повтори.

### «permission denied» при `npm install`
Добавь `sudo ` в начало команды: `sudo npm install -g firebase-tools`.
Mac спросит пароль.

### «Not currently logged in»
Повтори шаг 5 (`firebase login`).

### «No project is currently active»
Повтори шаг 7 (`firebase use --add`).

### «Function failed to deploy»
Прочитай последние 20 строк вывода — там будет причина. Скопируй их
мне, я помогу разобраться.

### Хочу начать заново с нуля
Ничего страшного. Это всё — просто настройки, никаких данных не
потеряется. Можно просто пройти шаги ещё раз.
